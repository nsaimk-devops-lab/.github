# nsaimk-devops-lab

## 🚀 Purpose

This organisation is a **hands-on DevOps & Cloud engineering lab** focused on solving **real-world infrastructure and platform challenges**, inspired by freelance job requirements and production scenarios.

Instead of building generic tutorial projects, this lab simulates how DevOps engineers actually work:
- Understanding ambiguous requirements  
- Designing scalable architectures  
- Making trade-offs under constraints  
- Delivering production-ready solutions  

---

## 🧠 Approach

Each project in this lab starts from a **realistic problem statement** (often inspired by freelance platforms like Upwork) and is treated as an engineering task rather than a simple implementation.

Every solution follows a structured workflow:

1. **Problem Definition**  
   Clear understanding of the challenge and goals  

2. **Assumptions**  
   Filling gaps where requirements are incomplete (as in real life)  

3. **Architecture Design**  
   System design with scalability, reliability, and cost in mind  

4. **Planning & Decisions**  
   Justifying technology choices and trade-offs  

5. **Implementation**  
   Infrastructure as Code, CI/CD pipelines, and deployments  

6. **Operations**  
   Monitoring, logging, and runbooks  

7. **Failure & Recovery Thinking**  
   Rollback strategies, risks, and postmortem analysis  

---

## 🧩 Project Structure

Each repository represents a **specific domain or technology area**, such as:

- `aws`
- `terraform`
- `kubernetes`
- `docker`
- `cicd`

Inside each repository, individual challenges are organised as:
```
project-name/
│
├── problem.md
├── assumptions.md
├── architecture.md
├── plan.md
├── decisions.md
│
├── infra/
├── cicd/
├── monitoring/
│
├── runbook.md
└── migration-strategy.md (optional)
└── rollback-strategy.md (optional)
```
